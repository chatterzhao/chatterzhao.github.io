一、什么是 Pages

hugo 是一个博客应用，写了博客肯定要对外发布，那用 Pages 就没错了

我说的不官方，将就者理解哈，它是一个免费的静态网站托管，我们可以用它来发布网站，免费的，大家实际用用理解就深了

二、如何配置 能同时提交代码到 GitHub 和码云

码云的 Pages 国内访问比较快

这里提到同时部署呢，是想丰富一下国内生态，码云是不错的选择，以前我不知道可以同时 push，就只提交一边，现在知道了，就两边同时提交了：

  • 取消 git config 用户名及邮箱的全局配置
  • 生成多个平台的密钥，各自上传到各自平台
  • 在项目git 下的config 中同时配置remote

更多请点这个两个连接：一台电脑生成多个密钥，同时提交多个平台

三、如何创建博客

简要：

  1. cd 到 blogs 目录，执行命令hugo new post/xxx.md新建文章文件，以 markdown 语法写完保存
  2. 如果要预览，cd 到 blogs 目录执行命令hugo server
  3. 使用主题编译成 html 文件进行发布，cd 到 blogs 目录执行命令hugo --theme=even
  4. cd 到 public 目录执行 git add .，git commit -m "说明本次提交改了什么",git push 这三个命令

我开始就是疑惑后面每篇文章怎么写，纠结了好几个小时，所以梳理了最后一段，供大家参考

详细：

我是参考这个网址:https://segmentfault.com/a/1190000012975914?utm_source=tag-newest

  1. 安装hugo：scoop install hugo，如果没有安装 scoop 点我了解
  2. 创建网站:cd 到需要创建项目的目录，或者在需要创建项目的目录，按住键盘 Shift 键，然后点右键，点击在此处打开 Powershell 窗口，然后执行这个命令hugo new site blogs，blogs自己随便写
  3. 创建后，现在还没有主题，也就是皮肤，我们到 hugo 官网去选择皮肤，就是 theme 菜单下，下载后，到目录 exampleSite 中复制，config.toml 这个文件，去 blogs 目录下替换原来的文件，然后这个文件里，可以进去看着改，很多是中文选项，更多皮肤安装操作看上面的参考链接
  4. 皮肤安装好后，我们使用 markdown 文件编辑工具，我这里是使用 VSCode，然后在blogs 目录下找到 content 目录，进去创建一个 post 目录，然后在里面协作 .md 格式的文件，比如我的第一篇博客.md，当然创建这个文件，最好是用命令，要命令，先 cd 到 blogs 目录，就是有 config.toml 文件的目录，它需要这个文件，然后执行命令hugo new post/我的第一篇博客.md
  5. 本地预览，cd 到 blogs 目录 执行命令hugo server，打开生成的连接，安装ctrl 鼠标点击，即可在浏览器打开
  6. 要发布到 Pages，这时post下的文件还没有在发布的网页里，需要执行命令：hugo --theme=even,even是我们的皮肤名字，如果你的不是，相应替换
  7. 现在到 public 目录执行git add .，git commit -m "说明本次提交改了什么",git push，现在到你的博客网址上就可以看到了，有时需要等个30秒左右才能刷新出新文章
  8. 参考文章里有自动化部署，大家可以尝试
